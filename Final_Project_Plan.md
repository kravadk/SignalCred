## Фаза 2 — Бекенд і зберігання

Налаштуй безкоштовний хостинг для бекенду і БД:

**База даних — Neon (https://neon.tech)**
Безкоштовний PostgreSQL, завжди активний, не засинає.
- Створи schema з усіма потрібними таблицями
- Підключи через `DATABASE_URL` в `.env`

**Бекенд — Railway (https://railway.app) або Render (https://render.com)**
Обидва мають безкоштовний tier який не засинає.
- Налаштуй всі ENV змінні
- Перевір що сервер відповідає на `GET /health`

**Фронтенд — Vercel (https://vercel.com)**
- Налаштуй rewrite rules: `/api/*` → бекенд URL
- Перевір що proxy працює

**Видали ВЕСЬ mock і локальний код:**
- Знайди кожне місце де є статичні або fake дані
- Замінити на реальні запити до бекенду
- Якщо бекенд недоступний — показувати skeleton або error state
- НІКОЛИ не показувати fake дані

## Фаза 3 — Виправлення всіх функцій

Пройдись по кожній вкладці і кожній кнопці.

Для кожного інтерактивного елементу перевір:
- Клікається → відправляє правильний запит → отримує відповідь → UI оновлюється
- Loading state показується поки запит виконується
- Error state показується якщо запит провалився + retry кнопка
- Success state правильно відображає результат

Якщо кнопка веде до blockchain транзакції:
- Перевір що wallet підключений перед викликом
- Перевір що правильна мережа (chainId)
- Показуй стан транзакції: `pending → confirmed / failed`
- Після успіху оновлюй UI без перезавантаження сторінки

## Фаза 5 — UI/UX виправлення

Перевір на всіх breakpoints: `360px / 768px / 1280px / 1440px`

1. Немає горизонтального скролу там де не повинно бути
2. Весь текст видимий — немає `overflow: hidden` який ріже текст
3. Кнопки не перекривають одна одну
4. Картки не виходять за межі контейнера
5. Шрифти завантажуються (немає FOUT / FOIT)
6. Зображення і іконки відображаються
7. Анімації не лагають і не мерехтять
8. Кольорова схема консистентна на всіх сторінках

Виправ кожну знайдену проблему.

## Фаза 8 — Оптимізація

**Performance:**
- Видали всі невикористані імпорти
- Lazy load важкі компоненти (charts, modals)
- Мемоізуй дорогі обчислення (`useMemo`, `useCallback`)
- Перевір bundle size (`npm run build` → перегляни output)

**Безпека:**
- Жодного API ключа у фронтенд коді
- `.env` в `.gitignore`
- Валідація всіх вхідних даних на беку
- Захист від дублікатів (унікальні ключі в БД)

**Надійність:**
- Всі async функції в `try/catch`
- Всі зовнішні API мають fallback або graceful error
- Webhook handlers відповідають < 200ms
- Немає memory leaks (`clearInterval`, `removeEventListener` при unmount)
